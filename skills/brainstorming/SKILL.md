# Brainstorming

Use this skill before any creative work — building features, creating components, adding functionality. Explore user intent, requirements, and design before implementing.

## When to use

- Before writing new functionality
- When requirements are ambiguous
- When multiple approaches are possible

## Process

1. Ask clarifying questions to understand the goal
2. Explore 2-3 alternative approaches
3. Identify trade-offs (complexity vs flexibility, speed vs maintainability)
4. Propose the most pragmatic approach
5. Get user confirmation before implementing
